# Configuration Cache

---

# Problem: Configuration Cache

<v-clicks>

🤦🏻 700KB cache entry size on 3 store views

📁 99.9% of uncached time spend on reading xml files

</v-clicks>

---

# Time spent on DB

<v-click>

<img src="/src/time-spend-on-db.png" class="ml-15 h-80%" />

</v-click>

---

# Better Approach

<v-clicks>

⚙️ Store each configuration scope as PHP file

📦 Loads data from DB for a scope as value override

</v-clicks>

---
layout: fact
---

# Layout Cache

---

# Problem: Layout Cache

<v-clicks>

🔑 Each unique entity has its own cache key for layout

🧩 Most of the layout is the same across the same entity type

📈 On large catalogs, cache can grow into 20Gbytes+ overtime

</v-clicks>

---

# Better Approach

<v-clicks>

⚙️ Store each merged layout handle scope as PHP include

📦 Loads data from DB for widgets

</v-clicks>

---
layout: fact
---

# Block HTML Cache

---

# Example: Price Block Cache

<v-clicks>

📈 Cache size grows with your catalog

🚫 Too volatile as it gets flushed on almost every product sale

</v-clicks>

---

# Better Approach

<v-clicks>

📥 Batch tier prices to eliminate N+1 query loops

🏷️ Batch catalog rule prices ahead of rendering

🧮 Aggregate min/final prices for child entities in configurable/bundle list items

</v-clicks>

---

# Example: Top Menu

<v-clicks>

⏲️ Slow generation because of an in-effective query

</v-clicks>

---

# Better Approach

<v-clicks>

⚙️ Move cache into Varnish by `ttl` attribute

🚀 Optimize a query to not use `LIKE '%/2/%`

</v-clicks>

# Full Page Cache

---

# Caveats: Full Page Cache

<v-clicks>

🎭 FPC is a mask that hides slow backend

🤖 Most of the pages in cache are generated by bots 

💥 Single product update has a huge blast radius

🧠 Personalization impact on Cache Hit ratio  

</v-clicks>

---

# Problem: ESI Double Rendering

<v-clicks>

🖼️ Rendering of ESI blocks still happens even when block is replaced with `<esi:include />` snippet

</v-clicks>

---

# Better Approach

<v-click>

🚫 Prevent rendering by around plugin on `renderNonCachedElement` of `Magento\Framework\View\Layout`

</v-click>
